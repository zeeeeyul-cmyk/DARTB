# 1️⃣ 개념정리

## 1-1. BigQuery 기본지식

~~~
✅ 학습 목표 :
* 데이터 관련 기초 지식(OLTP, SQL, Row, Column, 저장 형태 등)을 설명할 수 있다. 
* BigQuery 관련 기초 지식에 대해서 파악할 수 있다. 
~~~

이번 강의를 통해 데이터 분석을 위해 필요한 기본 개념과 BigQuery의 특징을 정리할 수 있었다. 먼저 OLTP는 주문, 결제, 회원가입처럼 실시간으로 데이터를 저장·수정·삭제하는 업무 처리 시스템이고, OLAP는 이렇게 저장된 데이터를 분석하는 데 초점이 맞춰진 시스템이라는 점을 배웠다.

또한 SQL은 데이터베이스에 저장된 데이터를 조회하고 가공하기 위한 언어이며, 원하는 조건에 맞는 데이터를 찾고 정리하는 데 사용된다는 점을 이해했다. 데이터는 보통 Row(행)와 Column(열)으로 구성되는데, Row는 하나의 기록을 의미하고 Column은 데이터의 속성을 의미한다.

이와 함께 BigQuery는 구글 클라우드 기반의 데이터 웨어하우스로, 대용량 데이터를 빠르게 분석할 수 있고 서버를 직접 관리하지 않아도 된다는 특징을 가진다는 점을 알게 되었다. 따라서 BigQuery는 초보자도 SQL을 활용해 데이터를 효율적으로 탐색하고 분석할 수 있는 도구라고 정리할 수 있다.



## 2-1. 데이터 활용 Overview

~~~
✅ 학습 목표 :
* 데이터를 활용하는 과정을 설명할 수 있다.
* 데이터를 탐색하는 과정으로 조건과 추출, 요약을 할 수 있다. 
~~~

데이터 활용은 단순히 데이터를 모아두는 것이 아니라, 문제를 정의하고 필요한 데이터를 수집한 뒤 저장하고, 탐색·분석하여 의사결정에 활용하는 과정이라는 점을 배웠다. 즉, 데이터를 활용한다는 것은 숫자를 보는 것 자체가 아니라 그 안에서 의미를 찾아내는 과*이라고 이해했다.

또한 데이터를 탐색할 때는 먼저 원하는 데이터를 조건에 맞게 골라내고, 필요한 항목만 추출하고, 이를 요약해서 전체적인 특징을 파악하는 과정이 중요하다는 것을 알게 되었다. 예를 들어 조건을 걸어 특정 데이터만 확인하고, 필요한 열만 선택하며, 개수·합계·평균 같은 집계를 통해 데이터를 요약할 수 있다.

이 과정을 통해 데이터를 바로 복잡하게 분석하기보다, 먼저 어떤 데이터가 있는지 파악하고 목적에 맞는 형태로 정리하는 것이 데이터 활용의 기본이라는 점을 이해했다. 공개된 강의 커리큘럼도 이 단원을 별도로 두고 있어, 기초 개념 뒤에 실제 활용 흐름을 익히도록 구성되어 있다.




## 2-2. 저장된 데이터 활용하기

~~~
✅ 학습 목표 :
* 데이터가 저장되는 형태를 알고 저장된 데이터를 활용할 수 있다. 
~~~

데이터베이스(Database) 는 데이터를 체계적으로 저장하고 관리하기 위한 시스템이다. 사용자는 데이터베이스를 통해 필요한 데이터를 저장하고, 수정하고, 조회할 수 있다. 즉, 데이터베이스는 데이터를 보관하는 기본적인 구조라고 볼 수 있다.

데이터 웨어하우스(Data Warehouse) 는 여러 곳에 저장된 데이터를 한데 모아 분석하기 좋은 형태로 관리하는 저장소이다. 일반적인 데이터베이스가 데이터의 저장과 운영에 초점이 맞춰져 있다면, 데이터 웨어하우스는 데이터를 분석하고 활용하는 데 더 적합한 구조를 가진다. 즉, 데이터 웨어하우스는 분석 목적의 데이터 저장 공간이라고 정리할 수 있다.

ERD(Entity Relationship Diagram) 는 데이터의 구조와 관계를 시각적으로 나타낸 도식이다. 어떤 테이블이 존재하는지, 각 테이블이 어떤 속성을 가지는지, 그리고 테이블끼리 어떤 관계로 연결되어 있는지를 한눈에 파악할 수 있도록 도와준다. 따라서 ERD는 저장된 데이터를 이해하고, 필요한 데이터를 정확히 조회하기 위한 기초 자료가 된다.

BigQuery는 이러한 데이터 활용 환경에서 대용량 데이터를 저장하고 분석할 수 있도록 지원하는 도구이다. 저장된 데이터를 활용하기 위해서는 먼저 데이터베이스와 데이터 웨어하우스의 차이를 이해하고, ERD를 통해 데이터 구조를 파악한 뒤, 필요한 데이터를 목적에 맞게 조회하는 과정이 필요하다.

## 문제 1

> **🧚Q. 포켓몬 게임이나 이커머스 산업과 같이 다양한 산업에서는 각기 다른 데이터가 존재합니다. 다음 중 하나의 산업을 선택하고, 해당 산업에서 수집하고 활용될 수 있는 데이터 항목 (칼럼) 5가지를 자유롭게 상상하여 나열해보세요.**
>
> - 예시 산업 
>
> >  온라인 음식 배달 / 스마트 헬스 케어 / 중고 거래 앱 / 교육 플랫폼 등 

<!--현실과 데이터 분석의 연결 고리를 상상하고, 데이터를 저장하는 형태를 활용하는 문제입니다. -->

<!--학습한 개념을 활용하여 자유롭게 설명해 보세요. 구체적인 예시를 들어 설명하면 더욱 좋습니다.-->

~~~
저는 교육 플랫폼 산업을 선택했습니다. 교육 플랫폼에서는 학습자의 강의 수강 기록, 학습 패턴, 성취도 등 다양한 데이터가 수집되고 활용될 수 있다고 생각합니다. 예를 들어 다음과 같은 데이터 항목을 저장할 수 있습니다.

1. 학생 ID
각 학습자를 구분하기 위한 고유한 값
2. 수강 강의명
학습자가 어떤 강의를 듣고 있는지 나타내는 정보
3. 강의 진도율
전체 강의 중 몇 퍼센트까지 학습했는지 보여주는 값
4. 문제 풀이 점수
퀴즈나 과제 수행 결과를 점수 형태로 저장한 값
5. 마지막 접속 일시
학습자가 언제 마지막으로 플랫폼에 접속했는지를 나타내는 정보

이러한 데이터는 단순히 저장하는 데서 끝나는 것이 아니라, 학습자의 학습 상태를 파악하고 맞춤형 서비스를 제공하는 데 활용될 수 있다. 예를 들어 진도율이 낮거나 마지막 접속일이 오래된 학습자를 확인하면 학습 독려 알림을 보낼 수 있고, 문제 풀이 점수를 바탕으로 학습자의 취약한 부분을 분석하여 복습 콘텐츠를 추천할 수도 있다.
즉, 교육 플랫폼에서는 다양한 데이터를 행과 열의 형태로 저장하고, 이를 통해 학습자의 행동을 분석하여 더 효과적인 학습 환경을 제공할 수 있다고 생각한다.
~~~



## 문제 2

> **🧚Q. 이번 강의를 통해 SQL이 왜 필요하다고 느끼는지, SQL을 통해 본인이 어떤 것을 해내고 싶은지를 자유롭게 작성해보세요.**

~~~
이번 강의를 통해 SQL은 저장된 데이터 중에서 내가 원하는 정보를 찾기 위해 필요한 언어라고 느꼈다. 데이터가 많이 쌓여 있어도 필요한 내용을 바로 알 수 있는 것은 아니기 때문에, 조건에 맞게 데이터를 조회하고 정리하는 과정이 중요하다고 생각했다. SQL은 이런 작업을 가능하게 해주는 도구라고 볼 수 있다.

또한 SQL을 사용하면 원하는 데이터만 골라서 확인할 수 있고, 데이터를 요약하거나 비교하는 것도 가능하다. 그래서 단순히 데이터를 보는 것보다, 필요한 정보를 더 정확하게 파악하는 데 도움이 된다고 느꼈다.

앞으로 SQL을 통해 데이터를 직접 조회해 보고, 그 안에서 필요한 정보를 찾는 연습을 해보고 싶다. 특히 마케팅이나 서비스 기획, 사용자 행동 분석과 관련된 데이터를 보면서 사람들의 이용 패턴을 파악하거나 서비스 개선 방향을 생각해보고 싶다. 이를 통해 데이터를 읽고 해석하는 능력을 키우고, 실제로 의사결정에 도움이 되는 분석을 해보고 싶다.
~~~
